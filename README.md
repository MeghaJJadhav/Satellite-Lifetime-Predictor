## 📌 Overview

This project builds an end-to-end machine learning pipeline to predict **how many days until a satellite re-enters Earth's atmosphere**, using publicly available Two-Line Element (TLE) orbital data from [Space-Track.org](https://www.space-track.org) and object metadata from the [ESA DISCOS](https://discosweb.esoc.esa.int) catalogue.

As of 2024, the U.S. Space Surveillance Network tracks over 27,000 catalogued objects in Earth orbit. Uncontrolled re-entries pose risks to active satellites, and inaccurate decay predictions waste propellant or cause mission failures. This project demonstrates that a data-driven approach — without access to real-time atmospheric density models — can achieve competitive predictive accuracy purely from orbital mechanics features extracted from TLE records.

## 🎯 Key Results

| Model | CV R² | CV MAE (days) | Validation Strategy |
|-------|--------|----------------|---------------------|
| Random Forest | **0.858** | **1.49** | GroupKFold (k=5) |
| XGBoost | **0.865** | **1.46** | GroupKFold (k=5) |

- **8,829** confirmed re-entry records processed (Space-Track.org, 2018–2024)
- **14-feature** per-satellite matrix engineered from 60-day TLE time-series
- **Group-aware cross-validation** — no NORAD ID appears in both train and validation folds

## ⚙️ Pipeline Overview

```
Space-Track.org API          ESA DISCOS CSV
      │                            │
      ▼                            ▼
 Decay Records              Object Class Metadata
 (PRECEDENCE=2)             (PAYLOAD / DEBRIS /
  8,829 satellites           ROCKET_BODY / MISSION_OBJECT)
      │                            │
      └──────────┬─────────────────┘
                 ▼
         TLE History Fetch
         (60-day window per satellite)
                 │
                 ▼
         Feature Engineering
         (14 features per satellite:
          mean motion, BSTAR drag,
          altitude stats, eccentricity,
          inclination, object class)
                 │
                 ▼
     ┌───────────────────────┐
     │  GroupKFold (k=5)     │
     │  cross-validation     │
     └───────────────────────┘
                 │
        ┌────────┴────────┐
        ▼                 ▼
  Random Forest        XGBoost
  (n=200 trees)     (n=500, lr=0.05)
        │                 │
        └────────┬────────┘
                 ▼
        Streamlit Dashboard
        (live prediction + feature importance)
```

## 🔬 Features Used

| Feature | Description |
|---------|-------------|
| `mean_motion_last` | Final orbital speed (rev/day) — encodes altitude via Kepler's third law |
| `mean_motion_std` | Variability in orbital speed — signals active drag evolution |
| `bstar_log_last` | Log-transformed drag coefficient at most recent epoch |
| `bstar_log_mean` | Average drag over 60-day window |
| `bstar_log_std` | Drag volatility — key indicator of imminent decay |
| `altitude_avg_last` | Mean altitude (periapsis + apoapsis / 2) |
| `altitude_range_last` | Orbital shape — higher range = more eccentric orbit |
| `ECCENTRICITY_last` | Orbits circularise before re-entry |
| `INCLINATION_last` | Latitude coverage affects atmospheric density exposure |
| `MEAN_MOTION_DOT_mean` | Average drag acceleration — direct measure of energy loss |
| `objectClass_*` | One-hot encoded object class from ESA DISCOS |

## 🛠️ Setup & Usage

### 1. Clone the repo
```bash
git clone https://github.com/YOUR_USERNAME/satellite-decay-predictor.git
cd satellite-decay-predictor
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Configure Space-Track credentials
The notebook fetches data via the authenticated Space-Track REST API. Create a free account at [space-track.org](https://www.space-track.org/auth/login) and set your credentials in the notebook's configuration cell:
```python
USERNAME = "your_email@example.com"
PASSWORD = "your_password"
```

### 4. Run the notebook
```bash
jupyter notebook Satellite_Decay_Prediction_final.ipynb
```
Run all cells top to bottom. This will generate `rf_model.pkl`, `xgb_model.pkl`, `feature_columns.json`, and `model_metrics.json`.

### 5. Launch the dashboard
```bash
streamlit run app.py
```

## 📉 Limitations

- Solar activity indices (F10.7 flux, Kp) are not included — a known driver of upper-atmospheric density variations
- 60-day TLE window may miss earlier drag evolution for objects with longer decay timescales
- Focused on LEO objects; higher orbital regimes are not covered

---

## 🔭 Future Scope

- Integrate NOAA space weather indices (F10.7, Kp/Ap) to improve accuracy during high solar activity
- Apply LSTM / Transformer architectures directly on raw TLE time-series
- Live NORAD ID polling mode — auto-fetch TLE history and return probabilistic re-entry window
- Extend prediction window to 90–180 days for longer-term deorbit planning
